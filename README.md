# BSAI.md v1.0

**Before Scaling with AI** is a portable Markdown method for reporting apparent document type, relative document scope, likely agent interpretation, evidence, confidence, ambiguity, and human decisions required before AI-assisted work scales.

BSAI does not recommend, authorize, approve, integrate, route, certify, or decide. It makes likely AI interpretation visible before a human, team process, governance process, or separate tool decides what happens next.

## Identity

`BSAI.md` is the method.

The examples show completed assessments, including one aligned-path assessment and three mismatch/risk assessments.

The template provides the assessment output format.

This release is intentionally Markdown-only. It is not a coding-agent skill, approval workflow, compliance process, or downstream integration layer.

## Use

Place `BSAI.md` in a repository root, documentation folder, project folder, prompt package, or review workspace.

Apply the method before AI-assisted work interprets project documents, generates tasks, generates code, merges artifacts, infers requirements, or treats project material as stronger than its apparent type and relative scope support.

## Core output

A BSAI assessment reports:

- declared type;
- apparent type;
- relative scope;
- type mismatch;
- scope mismatch;
- likely agent interpretation;
- likely over-interpretation risk;
- likely under-interpretation risk;
- confidence;
- evidence;
- counter-evidence;
- ambiguity;
- human decisions required.

## Boundary

BSAI is not the decision.

BSAI is the warning label, interpretation report, and scope check before the decision.
